Imagine I asked you to print "Hello World" 5 times. You could write console.log 5 times.
But what if I asked you to do it 1,000 times? Or what if you wanted to print every number from 1 to 100?

Writing that manually would be exhausting. 
Loops allow us to write a block of code once and tell the computer: 
"Repeat this until I tell you to stop."

The for Loop 🎡
The most common loop is the for loop. It needs three pieces of information to work:

Starting Point: Where do we begin? (e.g., let i = 0)

The Condition: When should we stop? (e.g., i < 5)

The Step: What happens after each round? (e.g., i = i + 1, which we usually write as i++)

Arrays (The "Super Box")

So far, we've put one thing in one box (like let name = "Ali"). But what if you have a list of things, like a grocery list or a list of high scores? 📝

Instead of creating 10 different variables, we use an Array. It looks like this:

JavaScript
let fruits = ["Apple", "Banana", "Cherry"];
An array is just one big box with compartments inside. Each compartment has a number, 
but here's the "noob trap": Computers start counting at 0. 

fruits[0] is "Apple"

fruits[1] is "Banana"

fruits[2] is "Cherry"

Loops + Arrays: The Power Couple
This is where things get really cool. We can use a for loop to go through every single item in an array automatically. Instead of writing a console.log for every fruit or color, we let the loop do the walking.

We can use array.length to tell the loop exactly when to stop, so it works no matter how many items are in the list.

Let's try it in VS Code
Add this to your index.js:

JavaScript
let students = ["Ali", "Sara", "Ahmed", "Zainab"];

for (let i = 0; i < students.length; i++) {
    console.log(`Hello ${students[i]}, welcome to the class!`);
}
