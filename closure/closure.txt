In JavaScript, a closure is a function bundled together 
with its lexical environment. It allows an inner function to "remember" 
and access variables from its outer (enclosing) scope even after the outer 
function has finished executing. 


Core Concept
Closures are created automatically every time a function is defined.
 Because JavaScript uses lexical scoping, a function's access to variables
  is determined by its physical location in the source code. 


Outer Function: Declares variables and an inner function.
Inner Function: References variables from the outer function.
Persistence: If the inner function is returned or passed elsewhere, 
it carries its "backpack" of outer variables with it. 


Practical Examples
Encapsulation (Private Variables): Use closures to hide data that
 should only be accessible through specific methods.
javascript
function createCounter() {
  let count = 0; // Private variable
  return {
    increment: () => ++count,
    getCount: () => count
  };
}
const myCounter = createCounter();
console.log(myCounter.increment()); // 1
Function Factories: Create specialized functions with preset parameters.
javascript
function makeAdder(x) {
  return (y) => x + y;
}
const add5 = makeAdder(5);
console.log(add5(10)); // 15
Asynchronous Code: Closures preserve state in setTimeout, event listeners,
 and API calls, ensuring the correct variables are available when the operation finally runs. 


Common Use Cases
Maintaining State: Preserving data between function calls without using global variables.
Module Pattern: Simulating private and public members for better code organization.
Memoization: Caching the results of expensive calculations within a private scope.
Currying: Transforming functions with multiple arguments into a sequence of functions.
